# Demo Feedback Themes

These notes simulate recurring repo feedback that instruction assets should address.

- Agents sometimes skip tests after small fixes and ship without verification.
- Responses are occasionally too verbose before they present actionable findings.
- Review outputs drift from the requested format and bury the main risks.
- Bug-fix work sometimes jumps into edits before reading the relevant files carefully.
- Skills should stay repo-specific and avoid broad unrelated scans.
